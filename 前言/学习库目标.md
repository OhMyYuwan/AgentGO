# 前言：学习库目标

欢迎来到 **Agent GO!**。

这个教程更适合刚进入 Agent 领域的新手，尤其是希望把 Agent 用在安全研究、代码分析、自动化工作流和落地应用开发中的同学们。我们并不假设读者已经完整理解 LLM、MCP、Skills、多 Agent 协作或项目级协议；相反，这个仓库就是希望从更实际的入口开始，带大家逐步认识这些概念，并在真实工具和真实项目中建立自己的判断。

需要先说明的是：本仓库不是一本封闭的教材，也不是某种标准答案。

它其实是我们长期实践后整理出来的一份**目录导向型教程**。里面会有章节说明、工具链接、项目案例、参考资料和我们自己的思考路径。我们希望读者不要只把它当成“读完就结束”的文档，而是顺着其中的链接去安装、去运行、去复现、去修改，再把这些经验重新整理成自己的知识体系和能力体系。

## 适合谁

这个学习库主要面向三类同学：

- 刚开始接触 Agentic Coding、Agent 框架和自动化工具的新手
- 想把 Agent 用在安全研究、漏洞分析、代码审计、情报整理等方向的同学
- 想用 Agent 辅助开发真实应用、搭建个人工具或推进团队工程效率的同学

如果你已经有一定开发经验，但还不知道 Agent 时代的工具链应该如何组织，这个仓库可以作为一张路线图。  
如果你刚开始学习编程或安全研究，也可以把它当成一个逐步进入实践场景的入口。

## 如何使用本仓库

建议不要一次性追求“看完所有章节”。更好的方式是：

1. 先读第一章，搭好基本环境，知道 IDE、CLI、MCP、Skill 和 Agent 框架大概各自站在哪里。
2. 按章节里的链接去安装和运行相关工具，不要只停留在概念理解。
3. 选择一个你感兴趣的实践项目，尝试复现它的功能和工作流。
4. 在复现过程中记录问题：哪里需要人工判断，哪里适合交给 Agent，哪里需要协议、记忆或工具支持。
5. 最后把这些经验整理成自己的方法，而不是机械照搬我们的配置。

Agent 领域变化很快，工具会更新，框架会迭代，最佳实践也会被不断推翻。因此，比记住某个具体命令更重要的是：理解一个 Agent 系统为什么需要入口、工具、技能、记忆、上下文和项目级协同。

## 我们的实践项目

这个教程也会逐步介绍我们在实践过程中沉淀下来的项目和想法，例如：

- **ProtoCodeBase**：围绕 ACP 协议展开的项目级协同平台，希望让软件可以像积木一样被 Agent 理解、复用和演化。
- **YuwanLabWriter**：面向写作、资料整理和内容生成的 Agent 应用探索。
- **TimeDiary**：围绕个人时间记录、生活数据和应用开发流程的实践项目。
- **TianGong.Skill**：通过引入 GAN 进行技能生成的评估与优化，不断锤炼一个技能直到其满足我们的评估标准。
- **ProtoCodeBase.Skill**：围绕ACP 协议展开的项目级协同与源码说明书实践。
- **Poison.Skill**：通过引入投毒机制阻止任何形式的个人“社会工程“级别的隐私泄露。

这些项目不是为了展示“我们做了一个多酷的 Demo”，而是希望通过真实项目说明：Agent 不只是聊天窗口里的问答能力，它可以参与软件工程、安全研究、知识整理和日常应用构建。

## 我们希望推动什么

这个仓库有两个目标。

一方面，它是给组内同学准备的快速上手路线。我们希望新人不用从零散资料里反复摸索，可以更快进入 Agent 工具链、项目实践和安全研究场景。

另一方面，我们也希望更多人能理解并认可我们的想法：Agent 应用不应该只停留在“让模型帮我写一段代码”，而应该继续向前走，走向可复用的 Skill、可协作的工作流、可继承的项目上下文，以及真正能被多人、多 Agent 持续推进的工程系统。

如果这个学习库能帮助你少走一点弯路，或者让你产生一个新的项目想法，那它就已经有意义了。
